1. Claude Code Instalação completa

Subagentes registrados globalmente, hooks automáticos e CLAUDE.md carregado em toda sessão.

  1. Cole a pasta no projeto

    estrutura 
    MeuProjeto/
├── Squad_Dev_v1.4/     ← copie aqui
├── src/
└── ...

  2. Abra o Claude Code e peça pra instalar

    bash 
    cd MeuProjeto
claude

    Diga ao Claude Code (uma única vez):

    prompt de instalação 
    Instale a Squad Dev v1.4 neste projeto.
A pasta Squad_Dev_v1.4/ já está aqui.
Leia Squad_Dev_v1.4/CLAUDE.md para entender o que precisa ser feito.

    Claude vai executar automaticamente:

    • Copiar .claude/commands/ para a raiz do projeto → 20 slash commands disponíveis imediatamente
    • Criar/mesclar .claude/settings.json com hooks (Prettier, ESLint, bloqueios, notificação)
    • Criar CLAUDE.md na raiz apontando para a Squad
    • Registrar os 10 subagentes globais em ~/.claude/agents/

    Por que funciona? Claude Code lê e escreve arquivos. O CLAUDE.md dentro da Squad descreve exatamente o que instalar — Claude usa suas ferramentas de arquivo para fazer isso sem scripts extras.

  3. Use — pipeline completo ou agente individual

    pipeline completo 
    /startcycle Sistema de agendamento online para clínicas
    agentes individuais 
    /architect  Projete a arquitetura do módulo de pagamentos
/database   Crie o schema multi-tenant com RLS
/backend    Implemente autenticação JWT + refresh token
/research   Qual a versão LTS do Node.js hoje?
/designhunter  Cace referências para SaaS B2B sério

API Keys & Variáveis de Ambiente

Segurança NUNCA commitar .env com valores reais. Adicione ao .gitignore antes de qualquer commit.

Crie app_build/.env com as variáveis necessárias para o seu projeto:

app_build/.env 
# LLM / AI
OPENAI_API_KEY=
ANTHROPIC_API_KEY=
GEMINI_API_KEY=

# Banco de Dados
DATABASE_URL=
POSTGRES_USER=
POSTGRES_PASSWORD=
POSTGRES_DB=

# Autenticação
JWT_SECRET=
JWT_EXPIRES_IN=7d
SESSION_SECRET=

# Cloud / Deploy
GCP_PROJECT_ID=
GCP_REGION=us-central1
CLOUD_RUN_SERVICE_NAME=

# Serviços Externos
SMTP_HOST=
SMTP_PORT=
SMTP_USER=
SMTP_PASSWORD=
STRIPE_SECRET_KEY=
STRIPE_WEBHOOK_SECRET=

# Aplicação
NODE_ENV=development
PORT=3000
APP_URL=http://localhost:3000
LOG_LEVEL=debug

Onde cada agente usa as variáveis

Variável Agente Como obter
DATABASE_URL @database-specialist, @backend-specialist Neon / Supabase / local
JWT_SECRET @backend-specialist openssl rand -base64 32
GCP_PROJECT_ID @devops-engineer Google Cloud Console

Em produção — use secrets manager

  • Cloud Run → gcloud secrets create MY_SECRET --data-file=.env
  • Docker / VPS → Docker Secrets ou .env fora do repositório
  • Vercel / Railway → Dashboard de variáveis de ambiente da plataforma

Checklist de setup

  • Criar app_build/.env (não versionado)
  • Preencher as variáveis necessárias para o projeto
  • Adicionar app_build/.env ao .gitignore
  • Configurar CI/CD com secrets manager
Nunca ❌ Commitar .env com valores reais · ❌ Hardcodar keys no código · ❌ Deixar keys em comentários · ❌ Compartilhar keys em chats

Dicas universais

Aprendizado por correção

Quando você corrigir o agente duas vezes sobre a mesma coisa, ou usar "sempre", "nunca", "toda vez", ele salva automaticamente a regra em AI_CONTEXT.md na seção "Regras Aprendidas". Você também pode pedir: "salva isso na memória".

Memória viva

Ao fim de cada fase, peça ao agente:

prompt 
Atualize Squad_Dev_v1.4/production_artifacts/memory/AI_CONTEXT.md
com stack, sprint atual, onde paramos e próximos passos.

Progressive disclosure

Não carregue todas as skills de uma vez. Carregue só a do agente ativo. Abra references/ quando precisar de detalhe específico.

Sessões curtas

Feche a sessão ao fim de cada fase, salve o contexto em AI_CONTEXT.md, abra nova sessão na próxima. Qualquer IA que ler AI_CONTEXT.md + sprints/sprint-N-context.md retoma exatamente de onde parou.

Security Gate antes de qualquer deploy

prompt 
Execute Squad_Dev_v1.4/.agents/workflows/sprint-security-gate.md no código atual
de app_build/ e entregue o Security_Gate_Report.md.
